## Processor Simulator

<Badge type="warning" text="Computer Architecture" />
<Badge type="info" text="C / C++" />
<Badge type="info" text="Assembly Concepts" />

A student computer architecture project focused on simulating how a simplified processor executes instructions. The goal is to connect architecture concepts with observable program behavior.

**Core ideas**

- Model instruction fetch, decode, execute, memory access, and write-back stages.
- Represent registers, memory, program counters, and instruction formats.
- Trace instruction execution step by step to make low-level behavior easier to understand.
- Use test programs to validate arithmetic, branching, and memory operations.

```txt
LOAD   R1, 100
ADD    R2, R1, R3
STORE  R2, 104
```

::: info Learning Outcome
This project strengthens the link between assembly language, processor design, and the way higher-level code is eventually executed by hardware.
:::

## Malware Analysis Toolkit

<Badge type="danger" text="Security" />
<Badge type="warning" text="Malware Analysis" />
<Badge type="info" text="Python" />
<Badge type="info" text="Static Analysis" />

A practical malware analysis toolkit concept for safely collecting indicators from suspicious files in a controlled lab environment.

**Planned and studied capabilities**

- Extract hashes such as MD5, SHA-1, and SHA-256 for sample identification.
- Inspect file metadata, imports, strings, and suspicious patterns.
- Generate compact analysis notes for repeatable investigation.
- Keep analysis workflows isolated from personal systems.

::: warning Safety Note
Malware analysis should be performed only in an isolated, controlled environment such as a dedicated virtual machine or sandbox.
:::
